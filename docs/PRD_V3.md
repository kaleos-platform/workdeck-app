# 쿠팡 광고 리포트 매니저 PRD V3

기준일: 2026-02-27

---

## 1. 개요 및 V2 대비 변경 범위

본 문서는 PRD V2(F001~F026)를 기준선으로, 대시보드 기간 필터 확장·예산/목표 설정 기능·관리 지표 추가 요구사항을 정의합니다.

### V3 변경 범위 요약

| 영역                         | 변경 내용                                                                   |
| ---------------------------- | --------------------------------------------------------------------------- |
| 인증                         | 이메일 인증 확인 링크 랜딩 → 로그인 리다이렉트 수정 (F027)                  |
| 대시보드                     | 기간 설정 추가 + 캠페인 목록 기간별 지표 및 이전 기간 대비 증감 표시 (F028) |
| 캠페인 상세 > 키워드 분석 탭 | 전체 키워드 표시 전환 + 필터 옵션 추가 (F029)                               |
| 캠페인 상세 > 대시보드 탭    | 일 예산 / 목표 ROAS 설정 CRUD + 이력 관리 (F030)                            |
| 캠페인 상세 > 대시보드 탭    | 일 예산 소진율 · 목표 ROAS 달성율 관리 지표 표시 (F031)                     |
| 캠페인 상세 > 대시보드 탭    | 예산/ROAS 변경 저장 후 메모 팝업 (F032)                                     |

### V2 기준선 (변경 없음)

F001~F026은 현행 구현 상태를 유지하며, V3 기능은 기존 기능을 확장·개선하는 방식으로 구현합니다.

## 2. 신규/변경 기능 명세

### F027 — 인증 메일 확인 링크 리다이렉트 수정

**관련 영역**: Supabase Auth 인증 메일 설정

**요구사항**

- 사용자가 가입 확인 이메일의 "인증 확인" 링크를 클릭한 후 이동하는 화면을 랜딩 페이지(`/`)가 아닌 로그인 화면(`/login`)으로 변경한다.
- 로그인 화면으로 이동 시 인증 완료 안내 문구를 표시한다.

**구현 상세**

- Supabase 대시보드 또는 `app/auth/callback/route.ts`에서 인증 완료 후 리다이렉트 경로를 `/login?verified=success`로 수정한다.
- F015에서 구현한 `verified=pending` 처리 로직을 활용하여 `verified=success` 파라미터일 때 "이메일 인증이 완료되었습니다. 로그인해주세요." 문구를 추가 표시한다.
- 기존 `app/auth/callback/route.ts` 리다이렉트 경로만 변경하면 된다.

**수락 기준**

- [ ] 인증 메일 링크 클릭 후 `/login` 페이지로 이동한다.
- [ ] `/login` 페이지에 "이메일 인증이 완료되었습니다." 안내 문구가 표시된다.
- [ ] 일반 로그인 진입 시에는 안내 문구가 표시되지 않는다.

### F028 — 대시보드 기간 설정 + 캠페인 목록 지표/증감 개선

**관련 페이지**: 대시보드 페이지 (`/dashboard`)

**요구사항**

- 대시보드 상단에 기간 설정 영역을 추가한다. 기본값은 최근 7일이다.
- 기간 변경 시 캠페인 목록의 지표 정보가 선택한 기간 기준으로 업데이트된다.
- 캠페인 목록에 기간별 지표(총 광고비, 평균 ROAS, 총 매출액)와 이전 동일 기간 대비 증감을 함께 표시한다.

**구현 상세**

기간 설정 UI

- 기존 캠페인 상세 화면의 `FilterBar` 컴포넌트(`src/components/dashboard/filter-bar.tsx`)의 날짜 범위 선택 기능을 재사용한다.
- 대시보드 페이지 상단(캠페인 목록 위)에 날짜 범위 피커를 배치한다.
- 기본 설정값: 오늘 기준 최근 7일 (`today - 6일 ~ today`)
- 선택 가능 단위: 7일, 14일, 30일, 직접 입력

캠페인 목록 지표 표시

- 각 캠페인 카드/행에 선택된 기간 동안의 지표를 표시한다.
  - 총 광고비 (원 단위 포맷)
  - 평균 ROAS (소수점 2자리, % 단위)
  - 총 매출액 (원 단위 포맷)
- 이전 동일 기간 계산 방식 (F021과 동일):
  - 현재 기간: `startDate ~ endDate` (N일)
  - 이전 기간: `startDate - N일 ~ startDate - 1일` (동일 N일)
- 증감 표시 형식:
  ```
  총 광고비
  28,841원
  ▲ +12.3%
  ```
- 증감 방향 색상: 광고비는 상승 시 빨간색, ROAS·매출액은 상승 시 녹색
- 이전 기간 데이터가 없을 경우 증감 표시 생략

API 변경

- 기존 `/api/campaigns` GET 엔드포인트에 `startDate`, `endDate` 쿼리 파라미터를 추가한다.
- 이전 기간 지표는 동일 API 응답에 `prevMetrics` 필드로 포함하거나, 클라이언트에서 별도 요청한다.

**수락 기준**

- [ ] 대시보드 상단에 날짜 범위 피커가 표시된다.
- [ ] 기본값이 최근 7일로 설정된다.
- [ ] 기간 변경 시 캠페인 목록의 지표가 재조회된다.
- [ ] 캠페인 목록에 총 광고비, 평균 ROAS, 총 매출액이 표시된다.
- [ ] 이전 동일 기간 대비 증감율이 색상 구분과 함께 표시된다.
- [ ] 이전 기간 데이터 없는 경우 증감 표시가 생략된다.

### F029 — 키워드 분석 탭 전체 키워드 표시 + 필터 옵션

**관련 페이지**: 캠페인 상세 페이지 > 키워드 분석 탭

**요구사항**

- 키워드 분석 탭의 기본 표시를 "광고비 또는 주문 수가 0인 키워드 제외" 방식에서 **전체 키워드 표시**로 변경한다.
- 기본 정렬 순서를 광고비 내림차순으로 변경한다.
- "광고비 및 주문 수 0인 키워드만 보기" 필터 옵션을 추가한다.
- "주문 발생 키워드만 보기" 필터 옵션을 추가한다.

**구현 상세**

기본 동작 변경

- 기존: 비효율 키워드(광고비 > 0, 주문 수 = 0)만 표시
- 변경: 전체 키워드 표시, 광고비 내림차순 정렬
- 기존 `/api/campaigns/[campaignId]/inefficient-keywords` API의 기본 필터 조건을 제거하거나, 새로운 범용 키워드 목록 API 추가를 검토한다.

필터 옵션 UI

필터 토글 버튼 또는 체크박스 형태로 다음 2가지 옵션을 제공한다:

| 필터 옵션                             | 조건                        | 기본값 |
| ------------------------------------- | --------------------------- | ------ |
| 광고비·주문 수 모두 0인 키워드만 보기 | `adCost = 0 AND orders = 0` | OFF    |
| 주문 발생 키워드만 보기               | `orders >= 1`               | OFF    |

- 두 필터는 동시에 활성화될 수 없으며, 하나를 선택하면 나머지가 비활성화된다.
- 필터가 OFF일 때는 전체 키워드를 표시한다.

정렬

- 기본 정렬: 광고비 내림차순
- 기존 컬럼 헤더 클릭 정렬 기능은 유지한다.

**수락 기준**

- [ ] 키워드 분석 탭 진입 시 전체 키워드가 표시된다.
- [ ] 기본 정렬이 광고비 내림차순이다.
- [ ] "광고비·주문 수 0 키워드만 보기" 필터 활성화 시 해당 조건의 키워드만 표시된다.
- [ ] "주문 발생 키워드만 보기" 필터 활성화 시 주문 건수 1개 이상인 키워드만 표시된다.
- [ ] 두 필터가 동시에 활성화되지 않는다.

### F030 — 캠페인 일 예산 / 목표 ROAS 설정 관리

**관련 페이지**: 캠페인 상세 페이지 > 대시보드 탭

**요구사항**

- 캠페인별로 일 예산과 목표 ROAS를 수동으로 입력/수정할 수 있다.
- 일자별로 값이 관리되며, 변경 등록 시 변경 일자부터 새 값이 적용된다.
- 입력값이 없는 경우 사용자가 직관적으로 입력을 인지하고 유도할 수 있는 UI를 제공한다.
- 대시보드 캠페인 목록(`/dashboard`)에 설정된 일 예산과 목표 ROAS를 표시한다.
- 광고 집행일 기간 내 이전 일자의 값도 직접 수정할 수 있다.

**구현 상세**

데이터 모델

`CampaignTarget` 신규 테이블을 생성한다. (데이터 모델 변경 사항 참조)

값 적용 로직 (effectiveDate 기반)

- 특정 날짜 D의 일 예산/목표 ROAS: `effectiveDate <= D`인 레코드 중 가장 최근 값
- 새 값 등록 시: 해당 effectiveDate 레코드 생성 또는 업데이트
- 이전 일자 직접 수정: effectiveDate를 과거 날짜로 지정하여 저장

입력 유도 UI (미설정 상태)

- 대시보드 탭 내 새 섹션(F031 관리 지표 컴포넌트)에서 미설정 상태를 강조하여 표시한다.
- 표시 예시: "📋 일 예산이 설정되지 않았습니다. [설정하기] 버튼을 클릭해 입력해주세요."
- [설정하기] 버튼 클릭 시 인라인 폼 또는 Dialog가 열린다.

설정 UI 구성

- 입력 폼 필드: 일 예산 (원, 숫자), 목표 ROAS (%, 숫자), 적용 시작일 (DatePicker, 기본값: 오늘)
- 저장 버튼 클릭 시 effectiveDate 기준으로 저장
- 이력 보기 옵션: 변경 이력을 날짜별로 테이블 형태로 표시하는 영역 제공 (간략 토글)

대시보드 캠페인 목록 표시

- F028에서 추가되는 캠페인 카드에 현재 유효한 일 예산과 목표 ROAS를 추가로 표시한다.
- 미설정 시 "-"로 표시한다.

API 목록

| 메서드   | 엔드포인트                                       | 설명                                 |
| -------- | ------------------------------------------------ | ------------------------------------ |
| `GET`    | `/api/campaigns/[campaignId]/targets`            | 전체 설정 이력 조회                  |
| `GET`    | `/api/campaigns/[campaignId]/targets/current`    | 현재 유효한 값 조회 (선택 날짜 기준) |
| `POST`   | `/api/campaigns/[campaignId]/targets`            | 새 값 등록 (effectiveDate 포함)      |
| `PATCH`  | `/api/campaigns/[campaignId]/targets/[targetId]` | 특정 이력 수정                       |
| `DELETE` | `/api/campaigns/[campaignId]/targets/[targetId]` | 특정 이력 삭제                       |

- [ ] 캠페인 상세 > 대시보드 탭에서 일 예산/목표 ROAS를 입력할 수 있다.

### F031 — 캠페인 관리 지표 — 일 예산 소진율 & 목표 ROAS 달성율

**관련 페이지**: 캠페인 상세 페이지 > 대시보드 탭

**요구사항**

- 대시보드 탭에 "광고 관리 현황" 컴포넌트를 기존 KPI 카드(총 광고비, 총 매출액 등) 상단에 추가한다.
- 해당 컴포넌트에 일 예산 평균 소진율과 목표 ROAS 평균 달성율을 표시한다.
- 설정된 기간 내에 예산/목표 ROAS가 없는 경우 미설정 안내 문구를 표시한다.

**구현 상세**

"광고 관리 현황" 컴포넌트 위치

- 캠페인 상세 > 대시보드 탭 내 기존 KPI 카드 그룹 위에 독립된 섹션으로 표시한다.
- 섹션 헤더: "광고 관리 현황" (가이드라인 역할을 명시하는 서브텍스트 포함)
- 컴포넌트 디자인: 구분이 명확한 카드 또는 배너 형태, 기존 KPI 카드와 시각적으로 분리

일 예산 평균 소진율 계산

- 조건: 조회 기간 내 effectiveDate 기반으로 일 예산이 설정된 날 기준
- 계산:
  ```
  일별 소진율 = 해당 일 광고비 / 해당 일의 유효 예산 × 100
  평균 소진율 = 예산 설정일 기준 일별 소진율의 평균
  ```
- 표시 형식: `XX.XX%` (소수점 2자리)
- 미설정 상태: "해당 기간에 설정된 일 예산이 없습니다."

목표 ROAS 평균 달성율 계산

- 조건: 조회 기간 내 목표 ROAS가 설정된 날 기준
- 계산:
  ```
  일별 달성율 = 해당 일 실제 ROAS / 해당 일의 유효 목표 ROAS × 100
  평균 달성율 = 목표 ROAS 설정일 기준 일별 달성율의 평균
  ```
- 표시 형식: `XX.XX%` (소수점 2자리)
- 미설정 상태: "해당 기간에 설정된 목표 ROAS가 없습니다."

API 변경

- 기존 `/api/campaigns/[campaignId]/metrics` API에 `budgetUtilization`(일 예산 평균 소진율)과 `roasAchievement`(목표 ROAS 평균 달성율) 필드를 추가한다.
- `CampaignTarget` 이력과 `AdRecord` 데이터를 조인하여 계산한다.

**수락 기준**

- [ ] 대시보드 탭 상단에 "광고 관리 현황" 컴포넌트가 기존 KPI 카드 위에 표시된다.
- [ ] 일 예산 소진율이 `XX.XX%` 형식으로 표시된다.
- [ ] 목표 ROAS 달성율이 `XX.XX%` 형식으로 표시된다.
- [ ] 날짜 필터 변경 시 두 지표가 재계산된다.
- [ ] 예산 미설정 시 "해당 기간에 설정된 일 예산이 없습니다." 문구가 표시된다.
- [ ] 목표 ROAS 미설정 시 "해당 기간에 설정된 목표 ROAS가 없습니다." 문구가 표시된다.

### F032 — 예산/목표 ROAS 변경 시 메모 팝업

**관련 페이지**: 캠페인 상세 페이지 > 대시보드 탭

**요구사항**

- 일 예산 또는 목표 ROAS를 변경하고 저장한 후 메모 저장 여부를 확인하는 팝업을 표시한다.
- 팝업에서 메모를 남길 날짜를 선택할 수 있으며, 기본값은 오늘 날짜이다.
- 사용자가 확인하면 해당 날짜의 `DailyMemo`에 변경 내용이 자동 기록된다.

**구현 상세**

팝업 구성

1. 제목: "변경 내용을 메모로 남기시겠습니까?"
2. 자동 생성된 메모 내용 표시 (수정 가능):
   - 일 예산 변경 시: `일 예산 변경: {이전 예산}원 → {변경 예산}원`
   - 목표 ROAS 변경 시: `목표 ROAS 변경: {이전 목표 ROAS}% → {변경 후 목표 ROAS}%`
   - 둘 다 변경 시 두 줄로 표시
3. 날짜 선택 DatePicker (기본값: 오늘)
4. 버튼: [메모 저장] / [건너뛰기]

메모 저장 규칙

- F024와 동일하게 기존 `DailyMemo` CRUD API(F005) 활용
- 해당 날짜에 기존 메모가 있을 경우 줄바꿈 후 내용 추가 (덮어쓰기 금지)
- 해당 날짜에 기존 메모가 없을 경우 새 메모 생성

**수락 기준**

- [ ] 예산/목표 ROAS 저장 후 메모 팝업이 표시된다.
- [ ] 팝업에 변경 내용이 자동 생성된 문구로 표시된다.
- [ ] 날짜 선택 DatePicker가 오늘 날짜를 기본으로 표시한다.
- [ ] 날짜를 변경하여 다른 날짜에 메모를 저장할 수 있다.
- [ ] [메모 저장] 클릭 시 해당 날짜 메모에 내용이 기록된다.
- [ ] 기존 메모가 있을 경우 덮어쓰지 않고 내용이 추가된다.
- [ ] [건너뛰기] 클릭 시 팝업만 닫히고 메모는 기록되지 않는다.

## 3. 데이터 모델 변경 사항

### 3-1. CampaignTarget 테이블 — 신규 생성

캠페인의 일 예산과 목표 ROAS를 날짜 기준으로 이력 관리하는 신규 테이블입니다.

```prisma
model CampaignTarget {
  id            String    @id @default(uuid())
  workspaceId   String
  campaignId    String
  effectiveDate DateTime  @db.Date  // 이 값이 적용되기 시작하는 날짜
  dailyBudget   Int?                // 일 예산 (원), null = 미설정
  targetRoas    Float?              // 목표 ROAS (%), null = 미설정
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  workspace Workspace @relation(fields: [workspaceId], references: [id])

  @@unique([workspaceId, campaignId, effectiveDate])
  @@index([workspaceId, campaignId, effectiveDate])
}
```

| 컬럼명          | 타입              | 설명                             |
| --------------- | ----------------- | -------------------------------- |
| `id`            | `String (UUID)`   | 고유 식별자                      |
| `workspaceId`   | `String`          | 워크스페이스 ID (→ Workspace.id) |
| `campaignId`    | `String`          | 캠페인 ID                        |
| `effectiveDate` | `DateTime (Date)` | 적용 시작일                      |
| `dailyBudget`   | `Int?`            | 일 예산 (원), null = 미설정      |
| `targetRoas`    | `Float?`          | 목표 ROAS (%), null = 미설정     |
| `createdAt`     | `DateTime`        | 생성 일시                        |
| `updatedAt`     | `DateTime`        | 수정 일시                        |

유니크 키: `(workspaceId, campaignId, effectiveDate)`

값 조회 쿼리 패턴 (특정 날짜 D에 유효한 값):

```sql
SELECT * FROM campaign_targets
WHERE workspace_id = ? AND campaign_id = ? AND effective_date <= D
ORDER BY effective_date DESC
LIMIT 1;
```
