# 장애 대응 가이드 (Runbook)

## 개요

프로덕션 환경에서 발생할 수 있는 주요 장애 3가지에 대한 대응 절차.
코드 변경 없이 운영 조치로 해결 가능한 내용을 중심으로 기술한다.

**로그 확인 위치:** Vercel 대시보드 → Project → Functions → 해당 함수 선택

---

## 1. 업로드 실패

### 1-1. 필수 컬럼 누락 오류

**증상 (사용자 관점)**

- 파일 업로드 버튼 클릭 후 주황색 경고 박스가 표시됨
- "필수 컬럼이 누락되었습니다 — 누락된 컬럼: [컬럼명]" 메시지

**원인 분석**

- 업로드한 파일이 쿠팡 셀러센터 광고 리포트 형식이 아님
- 리포트 다운로드 시 기간 또는 항목 설정 오류로 일부 컬럼 누락

**해결 절차**

1. 쿠팡 셀러센터 → 광고 관리 → 리포트 → "광고유형별 성과 보고서" 선택
2. 전체 기간 및 전체 항목을 포함하여 다운로드
3. 다운로드한 파일(`.xlsx` 또는 `.csv`)을 다시 업로드

**예방 조치**

- 리포트 다운로드 시 "항목 전체 선택" 옵션 사용 권장

---

### 1-2. 파일 파손 또는 지원되지 않는 형식

**증상 (사용자 관점)**

- 업로드 후 "업로드에 실패했습니다" 에러 토스트 메시지

**원인 분석**

- 파일이 `.xlsx`/`.csv` 확장자이지만 내부 데이터가 손상됨
- 인코딩 오류 (특히 CSV의 EUC-KR 인코딩)

**해결 절차**

1. 쿠팡 셀러센터에서 파일을 재다운로드
2. CSV 인코딩 문제인 경우 Excel에서 파일을 열고 `.xlsx`로 저장 후 업로드

**예방 조치**

- `.xlsx` 형식 사용을 권장 (CSV 대비 인코딩 문제 적음)

---

### 1-3. DB 연결 오류

**증상 (사용자 관점)**

- 업로드 후 수십 초 대기 후 "업로드에 실패했습니다" 에러

**원인 분석**

- Supabase DB 연결 타임아웃 또는 `DATABASE_URL` 환경변수 오류

**해결 절차**

1. Vercel 대시보드 → Functions 로그에서 오류 메시지 확인
2. `DATABASE_URL`이 Production 환경변수에 등록되어 있는지 확인
   - Vercel → Settings → Environment Variables
3. Supabase 대시보드 → Database → Connection string이 변경되지 않았는지 확인
4. 환경변수 수정 후 Vercel에서 재배포 트리거

**예방 조치**

- Supabase 프로젝트 설정 변경(비밀번호 초기화 등) 시 반드시 Vercel 환경변수도 업데이트

---

## 2. 인증 오류

### 2-1. 세션 만료 또는 로그인 루프

**증상 (사용자 관점)**

- 로그인 후 즉시 다시 `/login`으로 리다이렉트됨
- 새로고침해도 계속 로그인 화면으로 돌아옴

**원인 분석**

- 브라우저 쿠키 손상 또는 Supabase 세션 토큰 만료
- `proxy.ts` 미들웨어의 `updateSession()` 실패

**해결 절차**

1. 브라우저 개발자 도구 → Application → Cookies → 해당 도메인 쿠키 전체 삭제
2. 브라우저 캐시 삭제 후 재시도
3. 시크릿/프라이빗 창에서 재로그인 시도

**예방 조치**

- 세션은 1시간마다 자동 갱신됨 — 장기간 탭을 열어둔 경우 페이지 새로고침 필요

---

### 2-2. 구글 OAuth 콜백 실패

**증상 (사용자 관점)**

- "구글로 로그인" 클릭 후 구글 인증 완료했으나 에러 페이지 또는 `/login`으로 돌아옴

**원인 분석**

- Supabase의 Google Provider 설정 오류
- Google Cloud Console의 승인된 리다이렉트 URI 미등록

**해결 절차**

1. Supabase 대시보드 → Authentication → Providers → Google이 활성화되어 있는지 확인
2. Google Cloud Console → API 및 서비스 → 사용자 인증 정보 → OAuth 2.0 클라이언트 ID 확인
3. 승인된 리다이렉트 URI에 아래 항목이 포함되어 있는지 확인:
   ```
   https://[supabase-project-ref].supabase.co/auth/v1/callback
   ```
4. 누락된 경우 추가 후 10분 대기 (Google OAuth 전파 시간)

**예방 조치**

- Supabase 프로젝트 변경 시 Google Cloud Console도 함께 업데이트

---

### 2-3. 이메일 인증 링크 만료

**증상 (사용자 관점)**

- 회원가입 후 수신한 인증 메일 링크 클릭 시 "Invalid link" 또는 에러 메시지

**원인 분석**

- Supabase 이메일 인증 링크 유효기간(기본 1시간) 초과

**해결 절차**

1. `/signup` 페이지에서 동일 이메일로 재가입 시도 → 새 인증 메일 발송
2. 또는 `/login` 페이지 → "비밀번호 찾기"로 재설정 링크 발송

---

## 3. 데이터 누락

### 3-1. 업로드 이력은 있지만 캠페인 목록에 미표시

**증상 (사용자 관점)**

- 업로드 성공 메시지를 받았으나 `/dashboard` 캠페인 목록이 비어 있음

**원인 분석**

- 업로드된 데이터의 `campaignId`가 빈 문자열이거나 파싱 오류
- 날짜 파싱 실패 (지원되지 않는 날짜 형식)

**해결 절차**

1. Vercel Functions 로그에서 `/api/reports/upload` POST 응답 확인
   - `inserted: 0`이면 파싱된 행이 없음을 의미
2. 업로드한 파일의 '날짜' 컬럼 형식 확인 — `YYYYMMDD` 8자리 형식이어야 함
3. '캠페인 ID' 컬럼에 값이 존재하는지 확인
4. 파일 형식 확인 후 재업로드

**예방 조치**

- 업로드 후 "N개 행 저장 완료" 토스트의 N값을 파일 행 수와 대조 확인

---

### 3-2. 특정 기간 데이터 누락

**증상 (사용자 관점)**

- 일부 날짜의 데이터가 차트에 표시되지 않음
- 업로드 이력에는 해당 기간 업로드가 표시됨

**원인 분석**

- 해당 기간 재업로드 시 "중복 제외 저장"을 선택하여 기존 데이터 유지됨
- 또는 해당 날짜의 광고 집행이 실제로 없었던 경우

**해결 절차**

1. `/dashboard/upload`에서 해당 기간 파일 재업로드
2. 중복 데이터 다이얼로그에서 **"덮어쓰기"** 선택
3. 캠페인 상세 페이지에서 기간 필터로 해당 날짜 범위 재확인

**예방 조치**

- 동일 기간 데이터 수정이 필요한 경우 항상 "덮어쓰기" 선택
