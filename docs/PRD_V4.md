## 2. 신규/변경 기능 명세

### F033 — 메모 일자 수정 기능

**관련 컴포넌트**: `MemoDialog` (`src/components/dashboard/memo-dialog.tsx`)

**요구사항**

- 기존 메모를 수정할 때 날짜를 변경할 수 있다.
- 날짜 변경 후 저장하면 기존 날짜 메모는 삭제되고, 변경된 날짜에 내용이 저장된다.
- 변경된 날짜에 이미 메모가 있을 경우, 기존 메모 내용에 줄바꿈 후 내용을 이어 붙인다.

**현재 구현 상태**

- `MemoDialog`의 수정 모드에서 날짜 필드가 `<p>` 텍스트로만 표시되어 변경 불가.
- `isEditMode` 판단을 `!!initialContent`로 하기 때문에 수정 모드 구분은 별도 prop(`isEditing`)으로 변경 필요.

**구현 상세**

- 수정 모드에서도 날짜 Input을 표시하여 변경 가능하도록 수정.
- 저장 시 날짜가 변경된 경우:
  1. 기존 날짜 메모 삭제 (`DELETE /api/campaigns/[id]/memos?date=<original_date>`)
  2. 변경된 날짜 기존 메모 조회 (`GET /api/campaigns/[id]/memos?from=<new_date>&to=<new_date>`)
  3. 기존 메모 있으면 줄바꿈 후 내용 이어 붙여서 upsert, 없으면 신규 생성
- 날짜 변경 없이 저장 시: 기존 동작(해당 날짜에 upsert)과 동일.
- `MemoDialog` props에 원래 날짜(`originalDate`) 별도 전달하여 변경 여부 감지.

**수락 기준**

- [ ] 메모 수정 다이얼로그에서 날짜를 변경할 수 있다.
- [ ] 날짜 변경 저장 시 기존 날짜 메모가 삭제된다.
- [ ] 변경된 날짜에 기존 메모가 있으면 줄바꿈으로 내용이 합쳐진다.
- [ ] 날짜 변경 없이 저장 시 기존 동작과 동일하다.

### F034 — 캠페인 상세 일 예산/목표 ROAS 설정 섹션 UI 개편

**관련 컴포넌트**: `CampaignTargetSection` (`src/components/dashboard/campaign-target-section.tsx`)

**요구사항**

- 현재 "광고 관리 현황" 카드 내에 통합된 일 예산/목표 ROAS 설정 UI를 별도 카드로 분리한다.
- 분리된 카드명은 "일 예산 / 목표 ROAS"로 표시한다.
- 분리된 카드는 "광고 관리 현황" 카드 아래에 위치한다.
- 변경 이력은 이력 건수와 "[보기]" 버튼으로 표시하고, 클릭 시 팝업(Dialog)으로 이력을 표시 및 관리한다.

**현재 구현 상태**

- 단일 `CampaignTargetSection` 카드 안에 관리 지표(소진율·달성율) + 현재 유효 설정 + 이력 인라인 토글이 모두 통합되어 있음.

**구현 상세**

레이아웃 변경

```
[광고 관리 현황 Card]                  ← F031 (관리 지표만 남김)
  - 일 예산 평균 소진율
  - 목표 ROAS 평균 달성율

[일 예산 / 목표 ROAS Card]             ← F030 (신규 분리 카드)
  - 현재 유효 설정 (일 예산, 목표 ROAS, 적용 시작일)
  - [설정 추가] 버튼
  - [변경 이력 N건 보기] 버튼 → Dialog 팝업
```

이력 팝업 구성

- Dialog 형태로 표시
- 이력 목록 테이블 컬럼: 적용 시작일 | 일 예산 | 목표 ROAS | 수정/삭제 아이콘
- 각 행에서 수정 클릭 시 설정 수정 Dialog 열기 (기존 기능 재사용)
- 각 행에서 삭제 버튼 클릭 시 즉시 삭제
- Dialog 닫기 버튼 포함

**수락 기준**

- [ ] "광고 관리 현황" 카드에는 소진율·달성율 지표만 표시된다.
- [ ] "일 예산 / 목표 ROAS" 별도 카드가 "광고 관리 현황" 아래 표시된다.
- [ ] "변경 이력 N건 보기" 버튼이 표시된다. (이력이 없으면 버튼 미표시)

### F035 — 키워드 분석 필터 개선

**관련 페이지**: 캠페인 상세 > 키워드 분석 탭 (`app/dashboard/campaigns/[campaignId]/page.tsx`)

**요구사항**

- 필터 버튼 이름을 수정한다.
- 제거된 키워드를 숨기는 필터 토글을 추가한다.
- 필터 선택 시 정렬 순서를 자동으로 변경한다.

**현재 구현 상태**

- 필터 버튼명: `광고비·주문 수 0`, `주문 발생`
- 제거된 키워드(`removedAt !== null`)가 필터와 무관하게 항상 표시됨
- 필터 선택 시 정렬 순서가 자동 변경되지 않음

**구현 상세**

필터명 수정

| 현재             | 변경 후            |
| ---------------- | ------------------ |
| 광고비·주문 수 0 | 저효율 키워드 보기 |
| 주문 발생        | 주문 발생 키워드   |

제거된 키워드 숨기기 필터

- 추가 토글 버튼: "제거 제외"
- 기본값: OFF (제거 키워드 표시 — 현행 유지)
- 활성화 시: `removedAt !== null`인 키워드를 목록에서 제외
- 기존 필터("저효율 키워드 보기", "주문 발생 키워드")와 독립적으로 동작

필터 선택 시 자동 정렬 변경

- "저효율 키워드 보기" 선택 시: `kwSortBy = 'adCost'`, `kwSortOrder = 'desc'`
- "주문 발생 키워드" 선택 시: `kwSortBy = 'orders1d'`, `kwSortOrder = 'desc'`
- 필터 해제 시: 정렬 유지 (별도 초기화 없음)

**수락 기준**

- [ ] 필터 버튼명이 "저효율 키워드 보기", "주문 발생 키워드"로 표시된다.
- [ ] "제거 제외" 토글 버튼이 추가된다.
- [ ] "제거 제외" 활성화 시 removedAt 값이 있는 키워드가 목록에서 사라진다.
- [ ] "저효율 키워드 보기" 선택 시 광고비 내림차순으로 자동 정렬된다.
- [ ] "주문 발생 키워드" 선택 시 주문 수 내림차순으로 자동 정렬된다.

### F036 — 대시보드 탭 메모 팝업 자동 노출 버그 수정

**관련 파일**: `app/dashboard/campaigns/[campaignId]/page.tsx`

**재현 경로**

1. 캠페인 상세 > 대시보드 탭에서 성과 추이 차트의 날짜 클릭 → 메모 팝업 열림
2. 메모 팝업 닫기
3. "키워드 분석" 탭 또는 "광고 데이터" 탭 클릭
4. "대시보드" 탭으로 복귀
5. **버그**: 메모 수정 팝업이 자동으로 다시 열림

**기대 결과**: 대시보드 탭 재진입 시 메모 팝업이 자동 노출되지 않는다.

**원인 분석**

- 차트 날짜 클릭 → `memoTarget` 상태에 날짜 저장
- 탭 전환 후 대시보드 탭으로 복귀 시, `memoTarget`이 초기화되지 않은 채로 `DailyMemo` 컴포넌트가 다시 마운트·업데이트됨
- `DailyMemo`의 `useEffect`가 `targetDate` prop을 받아 팝업을 자동으로 여는 로직 실행

**구현 상세**

`handleTabChange` 함수에서 탭 변경 시 `memoTarget`을 초기화한다.

```typescript
function handleTabChange(nextTab: string) {
  if (nextTab !== 'dashboard' && nextTab !== 'keywords' && nextTab !== 'addata') return
  setMemoTarget(null) // 탭 변경 시 메모 타겟 초기화
  const params = new URLSearchParams(searchParams.toString())
  // ... 기존 URL 업데이트 로직
}
```

**수락 기준**

- [ ] 차트 날짜 클릭 → 다른 탭 이동 → 대시보드 탭 복귀 시 메모 팝업이 자동으로 열리지 않는다.
- [ ] 대시보드 탭에서 차트 날짜를 새로 클릭하면 메모 팝업이 정상 열린다.

### F037 — 대시보드 캠페인 목록 UI 개선

**관련 컴포넌트**: `CampaignListWithMetrics` (`src/components/dashboard/campaign-list-with-metrics.tsx`)

**요구사항**

- 일 예산과 목표 ROAS를 캠페인명 우측에 잘 보이도록 배치한다.
- 총 광고비, 평균 ROAS, 총 매출액의 증감 수치를 값 오른쪽에 나란히 표시한다.
- 캠페인 목록 너비를 전체적으로 균형있게 사용하도록 레이아웃을 개선한다.

**현재 구현 상태**

```
[캠페인명]
[adTypes]
[일 예산: X원 · 목표 ROAS: X%]    ← 좌측 하단 작은 텍스트

                        [총 광고비]  [평균 ROAS]  [총 매출액]
                         28,841원     XXX%         XXX원
                         ▲ +12%      (아래 별도 줄)
```

**변경 후 목표 레이아웃**

```
[캠페인명]  [adType 뱃지]
[일 예산 50,000원 · 목표 ROAS 300%]     [총광고비 28,841원 ▲+12.3%] [평균ROAS X% ▲+5%] [총매출액 X원 ▲+8%]
```

**구현 상세**

- 지표 영역: 각 지표별로 수치와 증감(`DiffBadge`)을 **한 줄에** 배치 (`flex items-center gap-1.5`)
- 레이아웃 비율: 캠페인 정보 영역 `flex-1 min-w-0`, 지표 영역 `shrink-0`으로 적절히 분배
- 일 예산·목표 ROAS: 기존 아래 줄 배치 유지하되 폰트/색상을 명확하게 개선

**수락 기준**

- [ ] 캠페인 카드에 일 예산과 목표 ROAS가 명확하게 표시된다.
- [ ] 총 광고비, 평균 ROAS, 총 매출액의 수치와 증감이 한 줄에 나란히 표시된다.
- [ ] 카드 레이아웃이 좌우 균형있게 구성된다.
