# 쿠팡 광고 리포트 매니저 MVP PRD v2 (통합본)

## 1. 제품 개요

- **목적**: 쿠팡 광고 리포트 Excel을 업로드·분석해 비효율 광고비를 절감하고 ROAS를 개선한다.
- **핵심 사용자**: 쿠팡 광고를 직접 운영하는 1인 셀러 또는 소규모 쇼핑몰 운영자.
- **MVP 가치**: 업로드 1회로 캠페인 성과 추이, 원시 데이터 탐색, 비효율 키워드 추출, 일자 메모 관리까지 수행한다.

## 3. 사용자 여정

```
1. 회원가입 페이지
   ↓ 이메일·비밀번호 입력 후 가입 완료

2. 워크스페이스 설정 페이지 (최초 1회)
   ↓ 쿠팡 사업자명 입력 후 생성

3. 대시보드 (전체 워크스페이스 요약)
   - 업로드 데이터 없음 → 리포트 업로드 CTA 표시
   ↓ 업로드 버튼 클릭

4. 파일 업로드 페이지
   - Excel(.xlsx) 파일 선택 또는 드래그앤드롭
   - 파싱 → DB 저장 → NVB 캠페인 목록 갱신
   ↓ 업로드 성공

5. 캠페인 상세 페이지 (NVB에서 캠페인 선택)
   - [대시보드 탭]    → 시계열 그래프 + 일자별 메모
   - [광고 데이터 탭] → 원시 데이터 테이블 (정렬·필터)
   - [키워드 분석 탭] → 비효율 키워드 목록 + 다중 선택 + 클립보드 복사
   ↓ 각 탭 상단의 공통 필터 조작

6. 날짜 범위·광고유형 필터
   - 모든 탭에 공통 적용, 필터 변경 시 해당 탭 데이터 즉시 갱신
```

## 4. 기능 명세 및 수락 기준

### 4.1 핵심 분석 기능

#### F001 광고 리포트 업로드

- **설명**: `.xlsx` 파일을 검증/파싱/정규화 후 `ReportUpload`, `AdRecord`를 저장한다.
- **수락 기준**:
  - 유효한 `.xlsx` 업로드 시 `inserted + updated > 0`을 반환해야 한다.
  - 유효하지 않은 확장자 업로드 시 `400 INVALID_FILE`을 반환해야 한다.
  - 동일 데이터 재업로드 시 업서트 키 기준으로 중복 행이 증가하지 않아야 한다.
  - 업로드 진행 중 프로그레스 바로 상태를 표시해야 한다.

#### F002 시계열 성과 그래프

- **설명**: 날짜별 `roas14d`, `adCost`, `clicks`, `impressions` 추이를 Recharts 차트로 표시한다.
- **수락 기준**:
  - 기간 필터 변경 시 차트가 즉시 갱신되어야 한다.
  - 데이터가 없으면 빈 상태 UI와 업로드 CTA를 표시해야 한다.
  - 지표 선택 토글(ROAS·광고비·클릭수·노출수 복수 선택)이 작동해야 한다.

#### F003 비효율 키워드 분석

- **설명**: `adCost > 0 AND orders1d = 0 AND keyword NOT IN ('-', '', null)` 조건의 키워드를 조회·복사한다.
- **수락 기준**:
  - 조건 일치 데이터가 있으면 테이블에 표시되어야 한다.
  - 체크박스 다중 선택(전체 선택 포함)이 가능해야 한다.
  - 선택 복사 시 쉼표(`,`) 구분 문자열이 클립보드에 저장되어야 한다.

#### F004 광고 데이터 테이블

- **설명**: 캠페인 원시 데이터를 정렬/페이지네이션으로 탐색한다.
- **수락 기준**:
  - `sortBy`, `sortOrder` 변경 시 일관된 결과를 반환해야 한다.
  - `pageSize`(10/25/50) 변경 시 응답 행 수가 반영되어야 한다.
  - 컬럼 표시·숨기기 토글이 작동해야 한다.

#### F005 일자별 메모 CRUD

- **설명**: 캠페인 단위 날짜 메모를 생성/조회/수정/삭제한다.
- **고유키**: `(workspaceId, campaignId, date)`
- **수락 기준**:
  - 동일 날짜 재저장 시 upsert로 1건만 유지해야 한다.
  - 삭제 후 재조회 시 해당 메모가 노출되지 않아야 한다.
  - 대시보드 탭 그래프와 연동되어 날짜별 메모를 확인할 수 있어야 한다.

### 4.2 공통 및 지원 기능

#### F006 날짜 범위 필터

- **설명**: 캠페인 상세 3개 탭에 공통 적용하는 날짜 범위 필터.
- **상태 정책**: URL Query를 단일 소스로 사용한다.
- **수락 기준**:
  - 탭 이동 시 필터 값이 유지되어야 한다.
  - 필터 변경 시 활성 탭 데이터가 즉시 재조회되어야 한다.

#### F007 광고유형 필터

- **설명**: 캠페인 상세 3개 탭에 공통 적용하는 광고유형 드롭다운 필터.
- **상태 정책**: URL Query를 단일 소스로 사용한다.
- **수락 기준**:
  - 탭 이동 시 필터 값이 유지되어야 한다.
  - 필터 변경 시 활성 탭 데이터가 즉시 재조회되어야 한다.

#### F010 기본 인증

- **설명**: Supabase Auth 기반 이메일/비밀번호 회원가입·로그인·로그아웃.
- **수락 기준**:
  - 유효한 이메일·비밀번호로 회원가입 후 워크스페이스 설정 페이지로 이동해야 한다.
  - 로그인 후 대시보드로 이동해야 한다.
  - 미인증 상태로 대시보드 접근 시 로그인 페이지로 리다이렉트해야 한다.

#### F011 워크스페이스 생성

- **설명**: 최초 가입 시 쿠팡 사업자명 기반 워크스페이스 1개 생성.
- **수락 기준**:
  - 사업자명 1자 이상 입력 시 워크스페이스가 생성되어야 한다.
  - 워크스페이스 미생성 사용자가 대시보드 접근 시 워크스페이스 설정 페이지로 리다이렉트해야 한다.
  - 1인 1워크스페이스 제한을 적용해야 한다.

├── 📤 리포트 업로드
│   └── 기능: F001 (Excel 파일 업로드·파싱·저장)

└── 📊 캠페인 목록
    ├── [캠페인명1] → 캠페인 상세 페이지
    ├── [캠페인명2] → 캠페인 상세 페이지
    └── ... (업로드된 Excel에서 파싱된 캠페인 자동 목록화)

## 7. 데이터 계약 (Excel → 정규화)

### 7.1 입력 컬럼 (핵심)

| 쿠팡 Excel 컬럼명 | DB 필드 |
|-----------------|---------|
| 날짜 | `date` |
| 광고유형 | `adType` |
| 캠페인 ID | `campaignId` |
| 캠페인명 | `campaignName` |
| 광고그룹 | `adGroup` |
| 광고집행 상품명 | `productName` |
| 광고집행 옵션ID | `optionId` |
| 광고 노출 지면 | `placement` |
| 키워드 | `keyword` |
| 노출수 | `impressions` |
| 클릭수 | `clicks` |
| 광고비 | `adCost` |
| 클릭률 | `ctr` |
| 총 주문수(1일) | `orders1d` |
| 총 전환매출액(1일) | `revenue1d` |
| 총광고수익률(1일) | `roas1d` |
| 총 주문수(14일) | `orders14d` |
| 총 전환매출액(14일) | `revenue14d` |
| 총광고수익률(14일) | `roas14d` |

### 7.2 정규화 규칙

- **날짜**: `YYYYMMDD` 또는 날짜 문자열을 `Asia/Seoul` 기준 날짜로 변환한다.
- **퍼센트**:
  - `"12.34%"` → `0.1234`
  - `"5.6689E-4"` → `0.00056689`
- **결측값**:
  - `"-"` 또는 빈 문자열은 컬럼 정책에 따라 `null` 또는 `0` 처리한다.
  - `keyword`의 `"-"`는 `null`로 정규화한다.
  - 수치 컬럼의 `"-"` 또는 빈 문자열은 `0`으로 처리한다.

### 7.3 업서트 키 정책 (최종)

- **복합 키**: `workspaceId + date + campaignId + adType + keyword + adGroup + optionId`
- 충돌 시 최신 업로드 값으로 갱신한다.
- "중복 날짜 병합" 정책은 사용하지 않는다.

## 9. API 계약

### `POST /api/reports/upload`

- **Request**: `multipart/form-data` (`file`: .xlsx)
- **Response**: `{ uploadId, inserted, updated, skipped, errors[] }`
- **Errors**: `400 INVALID_FILE`, `422 PARSE_ERROR`, `401 UNAUTHORIZED`, `500 INTERNAL_ERROR`

### `POST /api/workspace`

- **Request**: `{ name: string }`
- **Response**: `{ id, name, createdAt }`
- **Errors**: `400 INVALID_NAME`, `409 ALREADY_EXISTS`, `401 UNAUTHORIZED`

### `GET /api/campaigns`

- **Query**: `workspaceId`
- **Response**: `[{ campaignId, campaignName, adTypes[] }]`

### `GET /api/campaigns/:campaignId/metrics`

- **Query**: `from`, `to`, `adType=all|...`
- **Response**: `{ series: [{ date, adCost, clicks, impressions, roas14d }] }`

### `GET /api/campaigns/:campaignId/records`

- **Query**: `from`, `to`, `adType`, `page`, `pageSize`, `sortBy`, `sortOrder`
- **Response**: `{ items: [AdRecord], page, pageSize, total }`

### `GET /api/campaigns/:campaignId/inefficient-keywords`

- **Query**: `from`, `to`, `adType`
- **Response**: `{ items: [{ keyword, adCost, impressions, clicks, orders1d }] }`
- **필터 조건**: `adCost > 0 AND orders1d = 0 AND keyword NOT IN ('-', '', null)`

### `POST /api/campaigns/:campaignId/memos`

- **Body**: `{ date: string, content: string }` (upsert)
- **Response**: `{ id, campaignId, date, content, updatedAt }`

### `DELETE /api/campaigns/:campaignId/memos`

- **Query**: `date`
- **Response**: `{ success: true }`

| 상황 | 처리 방법 |

| 네트워크 오류 | 에러 토스트 + 재시도 액션 버튼 제공 |
