# 쿠팡 광고 리포트 매니저 MVP PRD v4 (현행화)

기준일: 2026-02-23

## 1. 프로젝트 핵심

- **목적**: 쿠팡 광고 리포트를 업로드해 광고비 낭비를 줄이고 성과 개선 의사결정을 빠르게 만든다.
- **타겟 사용자**: 쿠팡 광고를 직접 운영하는 1인 셀러/소규모 커머스 운영자.

## 2. 현재 구현 현황 요약

- **완료**: 인증, 워크스페이스 생성, 업로드/파싱/저장, 캠페인 상세 3개 탭, 메모 CRUD, 캠페인명 커스텀, 캠페인 삭제
- **부분 완료**: 광고유형 필터(F007)는 API 지원 완료, UI 노출은 현재 비활성화
- **미완료**: 자동화 테스트 체계(단위/통합/E2E), 운영 체크리스트 문서화, `/tasks` 운영 문서 체계

## 3. 사용자 여정

1. 랜딩 페이지에서 회원가입/로그인 선택
2. 인증 완료 후 워크스페이스가 없으면 워크스페이스 설정 페이지로 이동
3. 대시보드에서 리포트 업로드 진입
4. `.xlsx` 또는 `.csv` 업로드
5. 중복 데이터가 있으면 "덮어쓰기" 또는 "중복 제외 저장" 선택
6. 저장 완료 후 대시보드/캠페인 상세에서 분석
7. 캠페인 상세에서 기간 필터 변경, 탭 전환, 메모 기록, 키워드 복사

## 6. 페이지별 상세 기능

### 랜딩 페이지

- **역할**: 서비스 가치 소개 및 인증 진입점 제공
- **사용자 행동**: 회원가입/로그인 클릭
- **진입 조건**: 비로그인 상태 기본 진입
- **기능 목록**: 핵심 기능 소개, CTA 버튼
- **구현 기능 ID**: `F010`

### 로그인 페이지

- **역할**: 사용자 인증
- **사용자 행동**: 이메일 로그인 또는 Google OAuth 로그인
- **진입 조건**: 랜딩/리다이렉트
- **기능 목록**: 폼 검증, 인증, 대시보드 이동
- **구현 기능 ID**: `F010`

### 회원가입 페이지

- **역할**: 신규 계정 생성
- **사용자 행동**: 이메일 회원가입 또는 Google OAuth 가입
- **진입 조건**: 랜딩/로그인 페이지 링크
- **기능 목록**: 폼 검증, 계정 생성, 로그인 페이지 이동
- **구현 기능 ID**: `F010`

### 워크스페이스 설정 페이지

- **역할**: 최초 워크스페이스 생성
- **사용자 행동**: 사업자명 입력 후 생성
- **진입 조건**: 인증 완료 + 워크스페이스 미보유
- **기능 목록**: 이름 검증, 생성 API 호출
- **구현 기능 ID**: `F011`

### 대시보드 페이지

- **역할**: 워크스페이스 KPI 요약 및 캠페인 진입
- **사용자 행동**: 캠페인 선택, 업로드 페이지 이동
- **진입 조건**: 로그인 + 워크스페이스 보유
- **기능 목록**: 주간 KPI, WoW 표시, 캠페인 목록
- **구현 기능 ID**: `F001`, `F008`

### 리포트 업로드 페이지

- **역할**: 리포트 업로드/저장
- **사용자 행동**: 파일 선택, 중복 처리 방식 선택, 업로드 실행
- **진입 조건**: 대시보드 메뉴
- **기능 목록**: `.xlsx/.csv` 검증, 컬럼 검증 오류 안내, 중복 확인 다이얼로그
- **구현 기능 ID**: `F001`, `F013`

### 캠페인 상세 페이지 - 대시보드 탭

- **역할**: 시계열 성과 분석 및 메모 관리
- **사용자 행동**: 기간 변경, 지표 토글, 차트 클릭 메모 작성
- **진입 조건**: 사이드바 캠페인 클릭
- **기능 목록**: KPI 카드, 차트, 메모 테이블/다이얼로그
- **구현 기능 ID**: `F002`, `F005`, `F006`, `F008`, `F009`, `F012`, `F013`

### 캠페인 상세 페이지 - 키워드 분석 탭

- **역할**: 비효율 키워드 도출 및 복사
- **사용자 행동**: 정렬, 다중 선택, 복사
- **진입 조건**: 캠페인 상세 탭 전환
- **기능 목록**: 키워드 테이블, 체크박스, 복사
- **구현 기능 ID**: `F003`, `F006`, `F008`

### 캠페인 상세 페이지 - 광고 데이터 탭

- **역할**: 원시 데이터 탐색
- **사용자 행동**: 정렬, 페이지 이동, 컬럼 토글, 지면 필터
- **진입 조건**: 캠페인 상세 탭 전환
- **기능 목록**: 행 목록, 파생 컬럼, 계산 지표 컬럼
- **구현 기능 ID**: `F004`, `F006`, `F008`, `F013`

## 7. 데이터 모델

### User

- `id`, `email`, `name`, `avatarUrl`, `createdAt`, `updatedAt`

### Workspace

- `id`, `name`, `ownerId`, `createdAt`

### ReportUpload

- `id`, `workspaceId`, `fileName`, `uploadedAt`, `periodStart`, `periodEnd`

### AdRecord

- 핵심 필드: `workspaceId`, `reportId`, `date`, `adType`, `campaignId`, `campaignName`, `keyword`
- 성과 필드: `impressions`, `clicks`, `adCost`, `orders1d`, `revenue1d`, `roas1d`
- 확장 필드: `material`, `engagements`, `engagementRate`, `videoViews*`
- 유니크 키: `(workspaceId, date, campaignId, adType, keyword, adGroup, optionId)`

### CampaignMeta

- `workspaceId`, `campaignId`, `displayName`, `isCustomName`

### DailyMemo

- `workspaceId`, `campaignId`, `date`, `content`, `updatedAt`
- 유니크 키: `(workspaceId, campaignId, date)`

## 9. 구현 갭 및 정합성 이슈

- `F007` 광고유형 필터: API는 동작하지만 UI에서 `showAdTypeFilter={false}`로 비활성화 상태
- 테스트 체계: 저장소에 단위/통합/E2E 테스트 파일이 없음
- ROADMAP 참조 이슈: 기존 문서가 `/tasks` 경로를 참조하지만 실제 디렉토리가 없음
- 업로드 계약: 문서상 `inserted+updated` 모델과 실제 `중복확인/덮어쓰기/중복제외` 플로우가 상이했으며, 본 버전에서 현행화
