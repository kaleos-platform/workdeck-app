## 1. 개요 및 V1 대비 변경 범위

본 문서는 PRD V1(F001~F013, 완료된 MVP 현행 명세)을 기준선으로, 사용자 피드백 기반의 UX 개선 및 기능 보강 요구사항을 정의합니다.

### V2 변경 범위 요약

| 영역        | 변경 내용                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 랜딩 페이지 | 버튼 cursor:pointer 누락 수정 (F014)                                                                                                                                                                                                            |
| 인증        | 이메일 인증 안내 문구 추가 (F015), 로그인 실패 메시지 개선 (F016)                                                                                                                                                                               |
| 파일 업로드 | 다중 파일 업로드 + 진행률 UI (F017), 이력 처리 결과 표시 (F018), 에러 케이스 안내 (F019)                                                                                                                                                        |

| 캠페인 상세 | 이전 동일 기간 대비 KPI 표시 (F021), 광고 데이터 탭 페이지네이션 개선 (F022), 키워드 분석 탭 컬럼 변경 + 광고비 비중 (F023), 키워드 복사 완료 팝업 + 메모 자동 기록 (F024), 키워드 [제거] 상태 관리 (F025), 광고 데이터 탭 말줄임표 툴팁 (F026) |

### V1 기준선 (변경 없음)

F001~F013은 현행 구현 상태를 유지하며, V2 기능은 기존 기능을 확장·개선하는 방식으로 구현합니다.

## 2. 신규/변경 기능 명세

### F014 — 랜딩 페이지 버튼 커서 개선

**관련 페이지**: 랜딩 페이지 (`/`)

**요구사항**

- 랜딩 페이지의 "로그인", "무료 시작" 등 모든 클릭 가능한 버튼 및 링크 요소에 `cursor-pointer` CSS 클래스가 적용되어야 한다.
- 마우스를 버튼 위에 올렸을 때 커서가 손가락 모양으로 변경되어 클릭 가능 여부를 사용자가 즉시 인지할 수 있어야 한다.

**구현 상세**

- shadcn/ui `Button` 컴포넌트는 기본적으로 `cursor-default`를 사용할 수 있으므로, 랜딩 페이지 전용 CTA 버튼에 `cursor-pointer` 클래스를 명시적으로 추가한다.
- `<a>` 태그 또는 `<Link>` 컴포넌트로 렌더링되는 내비게이션 버튼도 동일하게 확인한다.
- 전역 CSS에 `button { cursor: pointer; }` 규칙 추가를 검토하여 재발을 방지한다.

**수락 기준**

- [ ] 랜딩 페이지의 "로그인" 버튼 위에 마우스를 올렸을 때 손가락 커서가 표시된다.
- [ ] 랜딩 페이지의 "무료 시작" 또는 CTA 버튼 위에 마우스를 올렸을 때 손가락 커서가 표시된다.
- [ ] 기타 모든 인터랙티브 버튼/링크에서 동일하게 동작한다.

### F015 — 회원가입 후 이메일 인증 안내

**관련 페이지**: 로그인 페이지 (`/login`)

**요구사항**

- 회원가입 완료 후 로그인 페이지로 리다이렉트 시, 이메일 인증 완료 요청 안내 문구를 표시한다.
- 안내 문구는 회원가입 직후 1회만 표시되며, 이후 일반 로그인 진입 시에는 표시되지 않는다.

**구현 상세**

- 회원가입 완료 후 `/login?verified=pending` 등의 쿼리 파라미터를 추가하여 로그인 페이지로 이동한다.
- 로그인 페이지에서 해당 쿼리 파라미터가 있을 경우, 페이지 상단 또는 폼 상단에 안내 배너/알림을 표시한다.
- 안내 문구 예시: "가입한 이메일을 확인하고 인증을 완료해주세요. 인증 완료 후 로그인이 가능합니다."
- shadcn/ui `Alert` 컴포넌트(variant: default 또는 info 계열)를 활용한다.
- 안내 배너는 사용자가 닫거나 로그인 성공 시 사라진다.

**수락 기준**

- [ ] 회원가입 완료 후 로그인 페이지 진입 시 이메일 인증 안내 문구가 표시된다.
- [ ] 일반 로그인 페이지 진입 시에는 안내 문구가 표시되지 않는다.
- [ ] 안내 문구가 명확하고 사용자가 다음 행동(이메일 확인)을 알 수 있다.

### F016 — 로그인 실패 에러 메시지 개선

**관련 페이지**: 로그인 페이지 (`/login`)

**요구사항**

- 로그인 실패 시 실패 원인에 따라 구분된 에러 메시지를 표시한다.
- 현재 표시되는 일반적인 에러 메시지를 원인별로 세분화한다.

**구현 상세**

Supabase Auth 에러 코드 또는 메시지를 파싱하여 다음 케이스별 안내 문구를 제공한다.

| 에러 케이스            | 표시 메시지                                                                    |
| ---------------------- | ------------------------------------------------------------------------------ |
| 이메일/비밀번호 불일치 | "이메일 또는 비밀번호가 올바르지 않습니다."                                    |
| 이메일 인증 미완료     | "이메일 인증이 완료되지 않았습니다. 가입한 이메일의 인증 링크를 확인해주세요." |
| 존재하지 않는 계정     | "등록되지 않은 이메일입니다. 회원가입을 진행해주세요."                         |
| 네트워크/서버 오류     | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."                     |
| 너무 많은 시도         | "너무 많은 로그인 시도가 감지되었습니다. 잠시 후 다시 시도해주세요."           |

- 에러 메시지는 폼 하단 또는 submit 버튼 위에 표시하며, shadcn/ui `Alert` 컴포넌트(variant: destructive)를 활용한다.
- 기존 `react-hook-form` + `Zod` 폼 검증 에러와 서버 응답 에러를 분리하여 처리한다.

**수락 기준**

- [ ] 잘못된 비밀번호 입력 시 "이메일 또는 비밀번호가 올바르지 않습니다." 메시지가 표시된다.
- [ ] 이메일 인증 미완료 계정으로 로그인 시 인증 안내 메시지가 표시된다.
- [ ] 에러 메시지가 사용자가 다음 행동을 취할 수 있도록 충분한 정보를 제공한다.

### F017 — 다중 파일 업로드 + 진행률 UI

**관련 페이지**: 리포트 업로드 페이지 (`/dashboard/upload`)

**요구사항**

- 파일 선택 시 여러 개의 `.xlsx` 파일을 동시에 선택할 수 있다.
- 선택한 파일들을 순차적으로 업로드 및 저장 처리한다.
- 전체 진행 상태(파싱 단계, 저장 단계)를 사용자가 실시간으로 확인할 수 있다.
- 업로드 진행 중 페이지 이탈을 방지하는 UI를 제공한다.

**구현 상세**

파일 선택 UI

- `<input type="file" multiple accept=".xlsx,.csv">` 로 다중 파일 선택 지원
- 선택된 파일 목록을 카드 또는 리스트 형태로 표시 (파일명, 파일 크기, 상태 배지)
- 개별 파일 제거 버튼 제공

진행률 표시 UI

- 업로드 시작 시 배경 화면을 dimmed(반투명 오버레이) 처리하여 진행 상태에 집중할 수 있도록 한다.
- 오버레이 내부에 진행 상태 모달을 표시한다.
- 진행 상태 모달 구성:
  - 전체 진행률 표시 (예: "3개 중 1번째 처리 중")
  - 현재 처리 중인 파일 이름 표시
  - 단계별 진행 상태 표시 (파일 파싱 중 → 중복 검사 중 → DB 저장 중 → 완료)
  - 저장 단계: 저장된 행 수 / 전체 행 수 표시
  - 완료: 결과 요약 표시 (저장 N건, 중복 제외 N건, 오류 N건)
  - "업로드 중에는 페이지를 이동하지 마세요." 안내 문구 표시
- 모든 파일 처리 완료 후 전체 결과 요약을 표시하고 모달 닫기 버튼 활성화

이탈 방지

- 업로드 진행 중 `beforeunload` 이벤트를 등록하여 브라우저 탭 닫기/새로고침 시 경고 표시
- 모달 배경 클릭 및 ESC 키로 닫기를 비활성화

상태 관리

- 파일별 처리 상태: `pending` | `parsing` | `uploading` | `saving` | `done` | `error`
- Zustand 또는 `useState` + `useReducer`로 관리

**수락 기준**

- [ ] 파일 선택 다이얼로그에서 여러 파일을 동시에 선택할 수 있다.
- [ ] 업로드 시작 시 배경이 dimmed 처리되고 진행 상태 모달이 표시된다.
- [ ] 각 파일의 처리 단계(파싱/저장)가 실시간으로 표시된다.
- [ ] 모달 내에 "업로드 중에는 페이지를 이동하지 마세요." 문구가 표시된다.
- [ ] 모든 파일 처리 완료 후 결과 요약이 표시된다.
- [ ] 처리 중 브라우저 탭 닫기 시도 시 경고 메시지가 표시된다.

### F018 — 업로드 이력 처리 결과 표시

**관련 페이지**: 리포트 업로드 페이지 (`/dashboard/upload`)

**요구사항**

- 업로드 이력 목록에 각 업로드의 처리 결과 정보(총 행 수, 저장된 행 수, 중복 행 수, 오류 행 수)를 함께 표시한다.

**구현 상세**

- `ReportUpload` 테이블에 처리 결과 컬럼을 추가한다. (데이터 모델 변경 사항 참조)
- 업로드 완료 시점에 해당 컬럼에 결과값을 저장한다.
- 업로드 이력 UI에서 각 이력 항목에 처리 결과 배지 또는 텍스트를 표시한다.
  - 표시 예시: "저장 245건 / 중복 제외 12건"
  - 오류가 있을 경우: "저장 230건 / 중복 제외 12건 / 오류 3건"
- 기존 이력(컬럼 추가 이전 데이터)은 처리 결과 없이 표시하며, `null` 처리를 통해 하위 호환성을 유지한다.

**수락 기준**

- [ ] 업로드 이력 목록에 처리 결과(저장 N건, 중복 N건)가 표시된다.
- [ ] 신규 업로드 시 처리 결과가 이력에 즉시 반영된다.
- [ ] 기존 이력(컬럼 추가 전)은 처리 결과 없이 정상적으로 표시된다.
- [ ] Prisma 마이그레이션이 정상 적용된다.

### F019 — 업로드 에러 케이스별 안내 문구

**관련 페이지**: 리포트 업로드 페이지 (`/dashboard/upload`)

**요구사항**

- 파일 업로드 시 발생할 수 있는 에러 케이스별로 구체적인 안내 문구를 표시한다.
- 파일 크기는 최대 10MB로 제한한다.

**구현 상세**

에러 케이스별 안내 문구:

| 에러 케이스                | 안내 문구                                                                                         |
| -------------------------- | ------------------------------------------------------------------------------------------------- |
| 파일 크기 초과 (10MB 이상) | "파일 크기가 10MB를 초과합니다. 파일을 분할하거나 용량을 줄인 후 다시 업로드해주세요."            |
| 지원하지 않는 파일 형식    | "`.xlsx` 또는 `.csv` 파일만 업로드할 수 있습니다."                                                |
| 필수 컬럼 누락             | "쿠팡 광고 리포트 형식과 맞지 않습니다. 올바른 리포트 파일을 선택해주세요. (누락 컬럼: {컬럼명})" |
| 빈 파일                    | "데이터가 없는 파일입니다. 내용을 확인 후 다시 업로드해주세요."                                   |
| 네트워크 오류              | "업로드 중 오류가 발생했습니다. 네트워크 상태를 확인 후 다시 시도해주세요."                       |
| 서버 처리 오류             | "서버에서 파일을 처리하는 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요."                     |

- 파일 선택 즉시(클라이언트 사이드) 파일 크기 및 형식을 검증한다.
- 파일 크기 제한은 클라이언트(`input` 검증)와 서버(API 라우트) 양측에서 적용한다.
- 서버 API에서도 10MB 초과 요청을 거부하고 명확한 에러 코드를 반환한다.
- 에러 메시지는 파일 목록 내 해당 파일 항목에 인라인으로 표시한다.

**수락 기준**

- [ ] 10MB 초과 파일 선택 시 즉시 에러 메시지가 표시된다.
- [ ] 지원하지 않는 파일 형식 선택 시 에러 메시지가 표시된다.
- [ ] 서버 API에서도 10MB 초과 파일을 거부한다.
- [ ] 각 에러 케이스별 안내 문구가 명확하고 사용자가 조치를 취할 수 있는 내용이다.

### F020 — 대시보드 WoW 증감 + 이전 수치 함께 표시

**관련 페이지**: 대시보드 페이지 (`/dashboard`)

**요구사항**

- 대시보드의 주요 지표 카드에 현재 주 수치와 함께 지난주 수치 및 WoW(주간 대비) 증감율을 함께 표시한다.

**구현 상세**

- 기존 WoW 증감율 표시에 더해, 이전 주의 실제 수치를 함께 노출한다.
- KPI 카드 레이아웃 예시:

  ```
  광고비
  28,841원
  ▲ +12.3%   (지난주: 25,040원)
  ```

- 표시할 지표: 광고비, 매출, ROAS, 클릭 수 등 기존 대시보드 KPI 카드 전체
- 이전 주 수치 포맷은 현재 주 수치와 동일한 형식(원, %, 건 등)으로 표시한다.
- 이전 주 데이터가 없을 경우 "이전 데이터 없음" 또는 해당 항목을 숨긴다.

**수락 기준**

- [ ] 대시보드 KPI 카드에 WoW 증감율과 함께 지난주 수치가 표시된다.
- [ ] 이전 주 데이터가 없는 경우 해당 항목이 graceful하게 처리된다.
- [ ] 수치 포맷이 현재 주와 일관성 있게 표시된다.

### F021 — 캠페인 상세 KPI 이전 동일 기간 대비 표시

**관련 페이지**: 캠페인 상세 페이지 > 대시보드 탭

**요구사항**

- 캠페인 상세 화면의 주요 지표 카드에 현재 조회 기간과 동일한 기간의 이전 구간 데이터를 비교하여 표시한다.
- 증감율과 이전 기간의 수치를 함께 표시한다.
- "이전 동일 기간 대비" 또는 유사한 레이블을 통해 사용자가 비교 기준을 명확히 인지할 수 있도록 한다.

**구현 상세**

이전 기간 계산 방식

- 현재 조회 기간: `startDate ~ endDate` (N일)
- 이전 비교 기간: `startDate - N일 ~ startDate - 1일` (동일 N일)
- 예시: 현재 2026-02-01 ~ 2026-02-14 (14일) → 이전 2026-01-18 ~ 2026-01-31 (14일)

KPI 카드 레이아웃 예시:

```
광고비
28,841원
▲ +15.2%  이전 동일 기간: 24,840원
```

표시 규칙

- 증감율: `(현재값 - 이전값) / 이전값 × 100`, 소수점 1자리 반올림
- 이전 기간 수치 표시: 총합 지표(광고비, 매출 등)는 총합, 비율 지표(ROAS, CTR, CVR 등)는 기간 평균
- 증감 방향에 따라 색상 구분: 긍정 지표 상승(ROAS, CTR 등) → 녹색, 하락 → 빨간색 / 광고비는 반대로 적용
- 이전 기간 데이터가 없을 경우 비교 표시 생략

API 변경

- 캠페인 상세 KPI API에 이전 기간 파라미터(`prevStart`, `prevEnd`) 또는 자동 계산 옵션을 추가한다.

**수락 기준**

- [ ] 캠페인 상세 KPI 카드에 "이전 동일 기간 대비" 레이블과 함께 증감율이 표시된다.
- [ ] 이전 기간의 수치(총합 또는 평균)가 함께 표시된다.
- [ ] 날짜 필터를 변경하면 이전 기간도 자동으로 재계산된다.
- [ ] 이전 기간 데이터가 없을 경우 비교 표시가 생략된다.
- [ ] 긍정/부정 지표에 따른 색상 구분이 직관적으로 적용된다.

- 현재 페이지 번호는 강조 표시(bold 또는 배경색 적용)

### F023 — 키워드 분석 탭 컬럼 변경 + 광고비 비중 표시

**관련 페이지**: 캠페인 상세 페이지 > 키워드 분석 탭

**요구사항**

- 키워드 분석 테이블의 표시 컬럼을 변경한다.
  - 제거: CVR, ROAS
  - 추가: 주문 수, 클릭 수, CPC, 광고비 비중
- 광고비 컬럼에 전체 광고비 대비 해당 키워드의 비중을 함께 표시한다.

**구현 상세**

컬럼 변경 내역:

| 기존 컬럼 | 변경 후                    |
| --------- | -------------------------- |
| CVR       | 제거                       |
| ROAS      | 제거                       |
| (신규)    | 주문 수 추가               |
| (신규)    | 클릭 수 추가               |
| (신규)    | CPC 추가                   |
| 광고비    | 광고비 + 비중(%) 함께 표시 |

CPC 계산 방식

- `CPC = 광고비 / 클릭 수`
- 클릭 수가 0인 경우 "-" 표시
- 소수점 1자리에서 반올림 (원 단위)
- 표시 예시: "1,245원"

광고비 비중 계산 방식

- `비중(%) = 해당 키워드 광고비 / 선택한 기간 전체 광고비 × 100`
- 전체 광고비는 현재 선택된 날짜 필터 기간의 캠페인 전체 광고비
- 소수점 2자리까지 표시(반올림)
- 표시 예시: "28,841원(15.23%)"
- 전체 광고비가 0인 경우 비중 "(0.00%)" 표시

변경 후 최종 컬럼 순서 (권장):

| 순서 | 컬럼명        | 비고                                  |
| ---- | ------------- | ------------------------------------- |
| 1    | 키워드        | 기존 유지                             |
| 2    | 제거 상태     | F025 신규 (제거된 키워드만 배지 표시) |
| 3    | 광고비 (비중) | 광고비 + 비중 %                       |
| 4    | 클릭 수       | 신규 추가                             |
| 5    | CPC           | 신규 추가                             |
| 6    | 주문 수       | 신규 추가                             |
| 7    | 노출 수       | 기존 유지                             |

**수락 기준**

- [ ] 키워드 분석 테이블에서 CVR, ROAS 컬럼이 제거된다.
- [ ] 주문 수, 클릭 수, CPC 컬럼이 추가된다.
- [ ] 광고비 셀에 "(비중%)" 형식으로 비중이 함께 표시된다.
- [ ] CPC는 `광고비 / 클릭 수`로 계산되며 소수점 1자리 반올림이 적용된다.
- [ ] 날짜 필터 변경 시 광고비 비중이 재계산된다.
- [ ] 클릭 수 0인 키워드의 CPC는 "-"로 표시된다.

## 3. 데이터 모델 변경 사항

### 3-1. ReportUpload 테이블 — 컬럼 추가

기존 `ReportUpload` 테이블에 업로드 처리 결과를 저장하는 컬럼을 추가합니다.

```prisma
model ReportUpload {
  // 기존 컬럼 유지
  id          String    @id @default(uuid())
  workspaceId String
  fileName    String
  uploadedAt  DateTime  @default(now())
  periodStart DateTime?
  periodEnd   DateTime?

  // V2 신규 추가 컬럼
  totalRows     Int?  // 파일 내 전체 데이터 행 수
  insertedRows  Int?  // 신규 저장된 행 수
  duplicateRows Int?  // 중복으로 제외된 행 수
  skippedRows   Int?  // 오류 등으로 건너뛴 행 수

  workspace Workspace @relation(fields: [workspaceId], references: [id])
}
```

마이그레이션 방식: `ALTER TABLE`로 nullable 컬럼 추가 (기존 데이터 영향 없음)

### 3-2. KeywordStatus 테이블 — 신규 생성

키워드의 [제거] 상태를 저장하는 신규 테이블입니다.

```prisma
model KeywordStatus {
  id          String    @id @default(uuid())
  workspaceId String
  campaignId  String
  keyword     String
  removedAt   DateTime?
  removedMemo String?
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  workspace Workspace @relation(fields: [workspaceId], references: [id])

  @@unique([workspaceId, campaignId, keyword])
}
```

| 컬럼명        | 타입            | 설명                             |
| ------------- | --------------- | -------------------------------- |
| `id`          | `String (UUID)` | 고유 식별자                      |
| `workspaceId` | `String`        | 워크스페이스 ID (→ Workspace.id) |
| `campaignId`  | `String`        | 캠페인 ID                        |
| `keyword`     | `String`        | 키워드 텍스트                    |
| `removedAt`   | `DateTime?`     | 제거 처리 날짜                   |
| `removedMemo` | `String?`       | 제거 관련 메모 내용 (참고용)     |
| `createdAt`   | `DateTime`      | 생성 일시                        |
| `updatedAt`   | `DateTime`      | 수정 일시                        |

유니크 키: `(workspaceId, campaignId, keyword)`
