## 개발 워크플로우

1. **작업 계획**: PRD 기능 ID(F014~F026)와 실제 코드 상태를 대조한다.
2. **작업 생성**: `tasks/` 디렉토리에 `XXX-설명.md` 문서를 작성한다. 각 문서에 목표·구현 단계·관련 파일·검증 기준을 포함한다.
3. **작업 구현**: 구조 → 데이터 → API → UI → 검증 순서로 진행한다.
4. **로드맵 업데이트**: 완료된 Task는 `✅ - 완료`로 표시한다.

  **대상 파일**: `app/(auth)/signup/`, `app/(auth)/login/`

- **Task 027 — F023: 키워드 분석 탭 컬럼 변경 + 광고비 비중**

  **대상 파일**: 캠페인 상세 > 키워드 분석 탭 컴포넌트, 키워드 API

  **구현 요점**
  - 기존 CVR, ROAS 컬럼을 제거하고, 주문 수/클릭 수/CPC/광고비 비중을 추가한다.
  - CPC = `광고비 / 클릭 수` (클릭 수 0이면 "-", 원 단위 반올림).
  - 광고비 비중 = `해당 키워드 광고비 / 기간 전체 광고비 × 100` (소수점 2자리).
  - 표시 예시: `28,841원(15.23%)`

  변경 후 최종 컬럼 순서:

  | 순서 | 컬럼명        | 비고                  |
  | ---- | ------------- | --------------------- |
  | 1    | 키워드        | 기존 유지             |
  | 2    | 제거 상태     | F025 신규 (배지 표시) |
  | 3    | 광고비 (비중) | 비중 % 함께 표시      |
  | 4    | 클릭 수       | 신규 추가             |
  | 5    | CPC           | 신규 추가             |
  | 6    | 주문 수       | 신규 추가             |
  | 7    | 노출 수       | 기존 유지             |

  **수락 기준**
  - [ ] CVR, ROAS 컬럼이 제거된다.
  - [ ] 주문 수, 클릭 수, CPC 컬럼이 추가된다.
  - [ ] 광고비 셀에 "(비중%)" 형식으로 비중이 함께 표시된다.
  - [ ] CPC 계산이 올바르고, 클릭 수 0인 경우 "-"로 표시된다.
  - [ ] 날짜 필터 변경 시 광고비 비중이 재계산된다.
